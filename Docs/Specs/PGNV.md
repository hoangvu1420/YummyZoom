## 1. Tên đề tài:
**Xây dựng hệ thống giao đồ ăn YummyZoom**

## 2. Lĩnh vực đề tài:
- **Lựa chọn 1:** Thương mại điện tử (eCommerce) và hậu cần (Logistic)
- **Lựa chọn 2:** Ứng dụng di động và web
- **Lựa chọn 3:** Phần mềm doanh nghiệp

## 3. Mục tiêu của ĐATN:

### 3.1. Kiến thức sinh viên thu thập được:
- Nghiên cứu Clean Architecture, thiết kế hướng miền (DDD), mẫu CQRS và phát triển API RESTful, ứng dụng đa nền tảng
- Thiết kế cơ sở dữ liệu, ORM, bộ nhớ đệm, xác thực JWT và phân quyền dựa trên claims
- Tích hợp các dịch vụ bên thứ ba như Stripe, Mapbox, Cloudinary và Firebase Cloud Messaging

### 3.2. Công nghệ sinh viên thu thập được:
- .NET Core 9, Entity Framework Core, PostgreSQL và .NET Aspire cho backend
- Flutter/Dart cho Android và Angular/TypeScript cho web quản trị
- Azure, Cloudinary, Vercel và các API như Stripe, Mapbox, FCM
- Docker, Azure Developer CLI và CI/CD pipeline cho DevOps

### 3.3. Kỹ năng sinh viên phát triển được:
- Phân tích, thiết kế hệ thống với 14 tập hợp miền và lập trình đa nền tảng
- Tối ưu hiệu suất, tích hợp và triển khai đa dịch vụ trên đám mây
- Quản lý dự án Agile và viết tài liệu kỹ thuật

### 3.4. Sản phẩm kỳ vọng:
- API backend hoàn chỉnh và hệ thống cơ sở dữ liệu được tối ưu với bộ nhớ đệm
- Ứng dụng di động Android cho khách hàng và web quản trị cho admin/nhà hàng
- Hệ thống sẵn sàng triển khai trên Azure cloud

### 3.5. Vấn đề thực tiễn đồ án giải quyết:
- Tạo nền tảng kết nối người tiêu dùng và nhà hàng với trải nghiệm tìm kiếm chuyên sâu và theo dõi đơn hàng thời gian thực
- Hỗ trợ nhà hàng quản lý tập trung và tích hợp thanh toán an toàn qua Stripe
- Giải quyết nhu cầu đặt món nhóm thông qua tính năng TeamCart

## 4. Các nội dung sẽ thực hiện và kế hoạch triển khai:

*Lưu ý: Khối lượng yêu cầu đối với đồ án tốt nghiệp hệ cử nhân là 6(0-0-12-12), tức là 12 tiết làm việc/tuần trong 17 tuần.*

### Nội dung 1: Tìm hiểu tổng quan về bài toán - từ Tuần **1** đến Tuần **3**
**Chi tiết:**
- Khảo sát các nền tảng giao đồ ăn hiện tại (GrabFood, ShopeeFood, Baemin) và phân tích yêu cầu người dùng
- Xác định các ca sử dụng chính và yêu cầu chức năng, phi chức năng của hệ thống
- Nghiên cứu Clean Architecture và thiết kế hướng miền (Domain-Driven Design)
- Tìm hiểu mẫu CQRS và kiến trúc hướng sự kiện

### Nội dung 2: Tìm hiểu tổng quan về công nghệ liên quan - từ Tuần **2** đến Tuần **6**
**Chi tiết:**
- Hệ sinh thái .NET Core với Web API, Entity Framework Core, Identity framework và SignalR
- Phát triển Flutter với Dart, hệ thống Widget và quản lý trạng thái
- Framework Angular với TypeScript, RxJS và Angular Material
- PostgreSQL với các tính năng nâng cao và tối ưu hiệu suất
- Dịch vụ đám mây Azure, Stripe API, Mapbox SDK, Cloudinary và FCM

### Nội dung 3: Phân tích thiết kế - từ Tuần **5** đến Tuần **9**
**Chi tiết:**
- Thiết kế 14 tập hợp miền chính và lược đồ cơ sở dữ liệu với mô hình đọc tối ưu
- Đặc tả API RESTful với luồng xác thực và thiết kế wireframes cho Android
- Thiết kế giao diện web quản trị và lập kế hoạch tích hợp dịch vụ bên thứ ba

### Nội dung 4: Xây dựng chương trình - từ Tuần **7** đến Tuần **15**
**Chi tiết:**
- Phát triển backend với các lớp Domain, Application, Infrastructure và triển khai PostgreSQL với EF Core
- Xây dựng ứng dụng Flutter cho Android và bảng điều khiển Angular cho quản trị
- Tích hợp Stripe, Mapbox, Cloudinary, FCM và xây dựng hệ thống xác thực JWT với claims

### Nội dung 5: Thử nghiệm và đánh giá - từ Tuần **14** đến Tuần **17**
**Chi tiết:**
- Kiểm thử toàn diện gồm unit tests, integration tests cho backend, mobile và web
- Kiểm thử hiệu suất với load testing và kiểm thử chấp nhận người dùng
- Triển khai hệ thống lên Azure, Vercel và phân phối APK

## 5. Lời cam đoan của sinh viên đã nhận được nhiệm vụ

Em xin cam kết sẽ hoàn thành các nhiệm vụ theo đúng kế hoạch đã đề ra, tuân thủ các yêu cầu kỹ thuật và thời gian quy định.

## PHỤ LỤC: ĐẶC ĐIỂM KỸ THUẬT NỔI BẬT

### **Tính năng đặc biệt của hệ thống:**
- **Tìm kiếm chuyên sâu:** Hỗ trợ tìm kiếm theo tên nhà hàng, tên món ăn, kiểu ẩm thực, và khoảng cách địa lý
- **TeamCart:** Tính năng đặt món nhóm với khả năng cộng tác thời gian thực
- **Theo dõi thời gian thực:** Cập nhật trạng thái đơn hàng và TeamCart liên tục
- **Đa nền tảng:** Ứng dụng Android (đầy đủ tính năng) + Ứng dụng web (tính năng giới hạn)

### **Kiến trúc kỹ thuật:**
- **Clean Architecture** với 14 tập hợp miền (domain aggregates)
- **Ủy quyền dựa trên khiếu nại** với quyền hạn được lưu trong bộ nhớ đệm
- **Tối ưu hiệu suất:** Bộ nhớ đệm, mô hình đọc, hàng đợi thông điệp
- **Đám mây gốc:** Triển khai Azure với điều phối .NET Aspire

### **Phạm vi sản phẩm tối thiểu khả thi (MVP):**
- Tập trung vào các tính năng nghiệp vụ cốt lõi
- Không bao gồm: Gợi ý AI, ứng dụng tài xế, phân tích nâng cao
- Quy mô nhỏ phù hợp với yêu cầu đồ án tốt nghiệp
